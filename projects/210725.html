Reservation Automation – Telegram Bot

A lightweight, automated reservation system designed for small food & beverage businesses that manage bookings through Telegram. This project simplifies reservation handling using Google Apps Script and Google Sheets to record, manage, and track customer bookings in real-time.

Webhook Integration

The bot is exposed via a secure HTTPS webhook endpoint in Google Apps Script. Incoming updates from Telegram are delivered to the `doPost(e)` function. The script parses the JSON payload, executes commands (/view <date>,/cancel <phone> <date>), and replies via the Bot API.

Objectives

The goal was to streamline the manual reservation process by reducing reliance on paper logs or inconsistent messaging. The project enables staff to add, cancel, or view reservations directly from Telegram using intuitive command-based inputs. All data is securely logged and organized by month inside Google Sheets.

Features

  • Flexible date inputs (e.g., today, 21/7, 7 Aug, Friday)
  • Country Code Handling.
  • Telegram Bot integration with Google Apps Script backend
  • Data stored and organised monthly in Google Sheets
  • Commands:
    • /view <date> – See reservations for a specific date
    • /cancel <phone> <date> – Cancel bookings

Outcome

The solution successfully automated daily reservation tracking with zero deployment cost. Reduce double-bookings and last-minute confusion, with a friendly command interface, minimal training required to adopt the system.

Future Enhancements

  • Dialogflow integration for natural language understanding - intent matching
  • Voice/call support for reservation via Twilio
  • Public-facing booking page for customers
  • Dashboard for reservation trends and analytics
